⏺ This is a personal expense tracker web app called "Spendly", built with Flask.
   It's a learning/tutorial project (note the "coming in Step X" placeholders)
  that will allow users to:

  - Register and log in to personal accounts
  - Add, edit, and delete expenses
  - View a profile page

  Currently only the landing, register, and login pages are functional. The
  remaining features (auth, expenses CRUD) are stubs waiting to be implemented
  across steps 3–9 of what appears to be a guided exercise.

❯ Explain the project structure to me                                           

  Searched for 1 pattern (ctrl+o to expand)

⏺ Here's the project structure (ignoring venv/, .git/, .mypy_cache/):

  expense-tracker/
  │
  ├── app.py                  # Entry point — Flask app, all route definitions
  │
  ├── database/
  │   ├── __init__.py         # Makes database/ a Python package
  │   └── db.py               # Placeholder for SQLite helpers (get_db, init_db,
   seed_db)
  │
  ├── templates/              # Jinja2 HTML templates
  │   ├── base.html           # Shared layout (navbar, footer) — all pages
  extend this
  │   ├── landing.html        # Home/marketing page
  │   ├── login.html          # Login form
  │   └── register.html       # Registration form
  │
  ├── static/
  │   ├── css/
  │   │   └── style.css       # All custom styles
  │   └── js/
  │       └── main.js         # Frontend JavaScript
  │
  ├── requirements.txt        # Python dependencies
  └── .gitignore

  Key relationships:
  - app.py is the single backend file — it creates the Flask app and registers
  all routes
  - templates/base.html is the master layout; other templates extend it using
  Jinja2 {% block %} tags
  - database/db.py is a stub — the SQLite connection logic hasn't been written
  yet (Step 1 of the exercise)
  - static/ files are served directly to the browser; referenced in base.html
